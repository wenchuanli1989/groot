### Application
- 每个应用固定有四个环境 `dev` `qa` `pl` `online` ，每个环境可以指定一个迭代版本 `Release`，例如当前迭代为 `v1.1.0`，线上环境为 `v1.0.0` ，那么应用的 `devRelease` `qaRelease` `plRelease` 就可以指定为 `v1.1.0` 的 `Release`，应用 `onlineRelease` 指向 `v1.0.0` 的 `Release`

### Release
- `archive` 为true组件实例不能在调整，迭代版本发布online环境时，自动设置为true

### ComponentVersion
- `publish` 表示是否可以对外公开使用，一旦发布禁止修改配置，并将该版本自动设置为组件的 `recentVersion` 

### ComponentInstance
- `trackId` 一般为组件实例第一次创建时的ID，多个版本迭代实例重新创建，但是trackI永远复制上一个版本的，保证多版本迭代之间还可以追溯组件实例的历史记录
- `parserType` 可视化平台兼容支持react和vue项目

### View
- `key` 实例英文名，如果是主入口，该值为页面访问地址
- `primaryView` 是否是主入口

### SolutionInstance
- `primary` 是否为主要解决方案，一个view可以有多个解决方案实例，但是只能有一个主要解决方案，其他解决方案只能提供公共组件给view使用

### 构建部署
- 部署时选择的环境迭代版本，自动设置为 `Bundle` 关联的迭代版本

### PropGroup
- `struct` Flat 没有配置块这一层，直接展示配置项，Default 直接下级为配置块，配置块直接下级为配置项

### PropBlock
- `layout` 布局方式
- `struct` 
- `listStructData` 

### PropItem
- `versionTraceId` 
- `valueOptions` 

### PropValue
- `abstractValueIdChain` 
- `valueStruct`
